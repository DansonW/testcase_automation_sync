# Test Case Automation Expert

這是一個專為「測試案例自動化生成」設計的專家技能。當使用者要求分析規格文件 (PDF/MD) 並產出測試案例 (Test Cases) 時，必須啟用此技能。

此技能封裝了 `GEMINI_SOP.md` 的標準作業程序與 `GEMINI_ERROR_LOG.md` 的避坑指南，確保產出的一致性與高品質。

## 何時使用 (When to Use)

*   當需要從規格文件（如 PDF, Markdown）產出測試案例時。
*   當需要分析 User Story 並轉化為具體測試步驟時。
*   當被要求執行 "Test Case Generation" 任務時。

## 核心職責 (Core Roles)

在執行任務期間，你必須同時扮演以下兩個角色：
1.  **資深軟體測試工程師 (Senior QA)**：專注於邊界值 (BVA)、等價類 (EP)、負面測試、權限與邏輯漏洞。
2.  **資深 UI/UX 工程師**：專注於介面回饋、排版佈局、防呆機制與操作流暢度。

## 執行流程 (Workflow)

你必須嚴格遵循以下順序執行任務：

### 第一階段：準備與避雷 (Preparation)
1.  **讀取錯誤紀錄**：在分析任何文件前，**必須**先讀取 `GEMINI_ERROR_LOG.md`。
2.  **載入檢查點**：將錯誤紀錄中的邏輯（如：特定廠商的免審核機制、欄位連動清空邏輯、CSV 跑版問題）載入你的短期記憶，作為審核規格的濾鏡。

### 第二階段：分析與生成 (Analysis & Generation)
1.  **建立目錄**：在 `generated_test_cases/` 下建立與來源資料夾同名的子目錄（跨資料夾任務使用 `System_Analysis`）。
2.  **產出 CSV**：根據規格撰寫測試案例，並嚴格遵守以下格式規範：
    *   **檔名格式**：`[來源資料夾]_test_case_[YYYYMMDD_HHMMSS].csv`
    *   **必填欄位**：`系統`, `權限`, `功能 / 頁面`, `測試功能`, `測試情境`, `操作步驟`, `期望結果`, `備註`。
    *   **格式強制**：
        *   所有欄位內容必須使用**雙引號 (`"`)** 包裹，以避免換行符號導致 CSV 跑版。
        *   `操作步驟` 與 `期望結果` 使用實際換行符號 (newline)，**禁用** `<br>`。
        *   `測試功能` 格式為 `【標籤】功能名稱` (標籤順序：權限 > 新增 > 正向 > 反向 > 整合 > 情境 > UI)。
        *   `備註` 僅填寫來源主檔名，不含章節。
    *   **內容排序**：權限 -> 合約 -> 一般功能 -> 報表/帳務。

### 第三階段：交付與同步 (Delivery & Sync)
1.  **儲存檔案**：將 CSV 寫入目標目錄。
2.  **執行同步**：呼叫 `upload_to_sheets.py` 將檔案上傳。
    *   指令：`python3 upload_to_sheets.py "generated_test_cases/[路徑]/[檔名].csv"`
3.  **驗證與回報**：確認上傳成功訊息，並向使用者回報「檔案已產出並同步至 Google Sheets」。

## 關鍵規則 (Critical Rules)

*   **系統別名**：若規格中出現 `OSS`，請視為 `ERP` 系統。
*   **不產出 UserStory 檔**：直接產出 Test Case CSV，省略中間的 User Story 分析檔。
*   **資安優先**：嚴禁在產出內容或執行過程中硬編碼 (Hardcode) 任何敏感資訊，所有設定優先讀取 `.env`。
*   **備註追溯**：若參考舊案例產出，`備註` 欄位必須回溯至最原始的 PDF 規格文件，而非舊的 CSV 檔名。
*   **權限命名規範**：使用標準化權限名稱，如：
    *   外部：`Media User`, `Media Admin`, `Agency User`, `Agency Admin`
    *   內部：`Onead User (PAD)`, `Onead User (PM)`, `Onead User (AOE)` 等。
*   **產出策略**：初期優先產出**單一維度** (如 `【權限】` 或 `【功能】`) 的測試項目，確認基礎覆蓋後，再產出複合維度案例。

## 錯誤檢查清單 (Error Checklist)

在產出最終檔案前，請自我檢查是否犯了以下錯誤（源自 Error Log）：
- [ ] 是否針對「果實夥伴 (Partners)」帳號處理了特殊 UI 與免審核邏輯？
- [ ] 是否檢查了「上刊後」的狀態鎖定（欄位唯讀）？
- [ ] 是否驗證了欄位連動（A 變更，B 需清空）？
- [ ] CSV 的每個欄位是否都正確使用了雙引號包裹？
- [ ] 測試功能標籤是否使用了全形括號 `【】`？
