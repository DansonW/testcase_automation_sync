項目概述

項目背景：
  • 在前一次改版受眾鎖定的基礎上，新增兩種受眾鎖定方式，以豐富產品的目標市場定位策略。

  • 廣告組合 之 受眾鎖定, 新增 產業 Shopper Graph 項目
  • 廣告組合 之 受眾鎖定, 新增 Webb 零售數據 項目

  • 受眾洞察報告 匯出時新增 產業 Shopper Graph 及 Webb 零售數據

項目範圍：
  • 廣告組合 之 受眾鎖定, 新增 UI/UX for 產業 Shopper Graph & Webb 零售數據
  • 受眾鎖定 產業 Shopper Graph & Webb 零售數據 能正常 建立 編輯
  • 受眾鎖定 所有鎖定項目組合, 能夠正常 建立 編輯
  • 受眾鎖定 所有鎖定項目組合, 能夠正常計算預估量體
  • 受眾洞察報告 能順利產出包含 產業 Shopper Graph & Webb 零售數據 且具有格式規範的報表
不包含：
  • 受眾洞察報告介面 UI/UX

4. [功能需求] 受眾鎖定 - 預估量體

原始頁面：https://guoshi.atlassian.net/wiki/spaces/ITO/pages/3848241164/- 
年齡 + 興趣特定議題 + 包含 ( 產業鑽石受眾 + 特定族群 + 產業 Shopper Graph + Webb 購買數據 + 網站流量受眾(已打包) + 廣告活動受眾(未打包)) + 排除 ( 網站流量受眾(未打包) + 廣告活動受眾(已打包))

step 1     Call API [ 年齡 & ( 產業鑽石受眾數 + 特定族群 + 產業 Shopper Graph + Webb 購買數據 + 網站流量受眾(已打包) ) ] = 150萬人
step 2    Call MASA [ 年齡 & 廣告流量受眾(未打包) ] = 5萬人
step 3    計算所有包含, 150萬人 + 5萬人 = 155萬人
step 4    使用者選取 年齡：男性 25-34, 女性 25-34｜取 DB(戶政司資料), 台灣人口數: 17,000,000 男性 25-34 比例 0.14, 女性 25-34 比例 0.13
step 5    17,000,000 * ( 0.14 + 0.13 ) = 4,590,000
上限值公式：min [戶政司人口數, 包含uu] * 興趣 ratio
step 6    min[4,590,000, 1,550,000] = 1,550,000
step 7    使用者選取 新聞時事, 娛樂追星, 體育運動｜取 DB(OneID 興趣比例資料), 新聞時事 比例 0.17, 娛樂追星 比例 0.12, 體育運動 比例 0.07
step 8    1,550,000 * ( 0.17 + 0.12 + 0.07 ) = 558,000 (上限值)
step 9    558,000 (上限值) * 0.85 = 474,300 (下限值)
step 10   Call MASA [ 年齡 & 網站流量受眾(未打包) ] = 3萬人
step 11   3萬人 * ( 0.17 + 0.12 + 0.07 ) = 10,800
step 12   Call API [ 年齡 &  廣告活動受眾(已打包) ] = 20萬人
step 13   20萬人 * ( 0.17 + 0.12 + 0.07 ) = 72,000
step 14   計算所有排除, 10,800 + 72,000 = 82,800
step 15   474,300 (下限值) - 排除uu 82,800 = 391,500
step 16   計算結果：391,500 ~ 558,000 UU
step 17   介面顯示：391,000 ~ 558,000 UU

7. [優化需求] 受眾鎖定 - 預估量體顯示

原始頁面：https://guoshi.atlassian.net/wiki/spaces/ITO/pages/3848208402/-
  • 動機：若僅選擇網站受眾包進行數據鎖定，實際量體計算結果很可能低於10,000（台灣擁有如此大流量的網站並不多）。當結果少於10,000時，系統會顯示0-0 UU，這可能導致使用者產生不安感，無法確定是受眾包本身有問題還是設定出現錯誤，故作提示文本修改。
  • 廣告活動之數據鎖定預估量體計算，當上限值小於 1 萬時
    • 目前：量體預估區間：0 - 0 UU
      你的廣告受眾太過特定，可能影響廣告投放，建議將受眾目標設定較廣泛。
    • 期望：量體預估區間：< 10,000 UU
      你的廣告受眾太過特定，可能影響廣告投放，建議將受眾目標設定較廣泛。
  • 若出現警語，依照目前系統架構，可能會導致使用者廣告無法完全投放的狀況。此為已知現象，目前不做任何處置。
