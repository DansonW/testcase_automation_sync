資料計算邏輯

數據計算

  1. 依據 version 將 daily row data 區分。
    範例：id = 24999，日期在 2020‑11‑17 ~ 2020‑11‑18，interests in [601, 602]，其他皆為 0（others）；所有項目皆為 0 的資料直接剔除。
  2. 將該 version 的 daily row data 進行權重計算，將 others 數值依權重分配給 interests。
    權重計算公式：FLOOR(all_dataset * dataset / sum(dataset))
    範例（cv）：
    ig 601 cv = FLOOR((9968+9912+275) * 9968 / (9968+9912)) = 10105
    ig 602 cv = FLOOR((9968+9912+275) * 9912 / (9968+9912)) = 10049
    id ig cv
    24999 601 9968
    24999 602 9912
    24999 0 275
  3. 依據 version 將 group row data 區分並加總，同 id 範例如下：
    id cv
    24999 42597
  4. 將 group row data 數值依權重分配給 daily row data：
    權重計算公式：FLOOR(group_dataset * daily_dataset / sum(daily_dataset))
    範例：
    ig 601 cv = FLOOR(42597 * 10105 / (10105+10049)) = 21357
    ig 602 cv = FLOOR(42597 * 10049 / (10105+10049)) = 21239
  5. 權重計算捨去的差異值，加到該 version 相對應項目的最小值：
    Min(cv) + (group_dataset - sum(權重計算結果))
    多筆最小值時，依 ig 排序小→大依序補 +1，直到差異值為 0。
  6. 將所有 version 權重計算結果依 id & ig & 指標（如 cv）加總，作為 API response。

daily row data SQL 範例：

SELECT 
  plaid,
  day_local as report_date,
  sumIf(counts, event='cv') as cv,
  sumIf(counts, event='impression') as impression,
  sumIf(counts, event='videoClick') as videoClick,
  sumIf(counts, event='bannerClick') as bannerClick,
  sumIf(counts, event='q0') as q0,
  sumIf(counts, event='q25') as q25,
  sumIf(counts, event='q50') as q50,
  sumIf(counts, event='q75') as q75,
  sumIf(counts, event='q100') as q100,
  sumIf(counts, event='view2s') as view2s,
  sumIf(counts, event='view3s') as view3s,
  sumIf(counts, event='view5s') as view5s,
  sumIf(counts, event='view10s') as view10s,
  sumIf(counts, event='view30s') as view30s,
  sumIf(counts,event='tb') as tb,
  sumIf(counts,event='eng') as eng
FROM kafka.summing_day_event_dicts
WHERE plaid in (40882) and day_local between '2022-09-22' and '2022-09-30'
GROUP BY plaid,day_local
ORDER BY plaid,day_local

資料計算邏輯

數據計算

  • 依 version 區分 daily row data：id=40882，日期 2022‑09‑22~2022‑09‑23，categories in [2,5,20,21]，其它為 0；全 0 列剔除。
  • 將該 version daily row data 進行權重，others 依權重分配至 categories。
    權重計算：FLOOR(all_dataset * dataset / sum(dataset))
    範例（cv）：
    categories_id 2 cv = FLOOR((9968+9912+275) * 9968 / (9968+9912)) = 10105
    categories_id 5 cv = FLOOR((9968+9912+275) * 9912 / (9968+9912)) = 10049
    id categories_id cv
    40882 2 9968
    40882 5 9912
    40882 0 275
  • 依 version 區分 group row data，依 id 加總：
    id cv
    40882 42597
  • group row data 與 daily row data 進行權重分配：
    權重：FLOOR(group_dataset * daily_dataset / sum(daily_dataset))
    範例：
    categories_id 2 cv = FLOOR(42597 * 10105 / (10105+10049)) = 21357
    categories_id 5 cv = FLOOR(42597 * 10049 / (10105+10049)) = 21239
  • 權重差異值補回：對該 version 對應指標最小值加差，若多筆最小則依 categories_id 排序小→大依序 +1，直到差為 0。
  • 將所有 version 的結果依 tier_id & categories_id & 指標加總，作為 API response。

資料計算邏輯（以 cv 為例）

group segment row data cv：

plaid sg_id cv
57666 3eef5... 116,790

group event row data cv：

57666 239,456

Step1. 權重計算公式

((segment_row_data / sum(segment_row_data)) * group_event_row_data).floor
範例：sg_id 3eef...
((116,790 / 573,738) * 239,456).floor = 48,743

Step2. 計算差異值並加回指定 segment

sum(cv 權重) = 48,743 + 35,951 + 49,367 + 52,799 + 52,591 + 1 = 239,452
差異 = 239,456 − 239,452 = 4；找出最小值 1，將 4 補上 => 5。
若有多個最小值，依 event key 排序規則逐一比較（event key 順序：[cv, impression, q0, view2s, view3s, view5s, view30s, q25, q50, q100, bannerclick, videoclick]）。
若仍無法唯一決定，則依 sg_id.sort 取第一筆。

Step3. 回傳結果

將所有 segment 權重後結果依 id & sg_id & 指標加總，作為 API response。

整體計算步驟摘要

  1. 將 group_segment_row_data 整為： 
    {
  cv: [116790, 86141, 118286, 126508, 126010, 3],
  impression: [79053, 56624, 79705, 86690, 86218, 2],
  ...
}
  2. 將 group_event_row_data 整為： 
    {
  cv: 239456,
  impression: 166717,
  ...
}
  3. 將上述兩者以 event 為 key 執行權重公式，segment_row_data=0 時補 0，不計算。
    結果結構類似： 
    {
  segment_A: { cv: 48743, impression: 33942, ... },
  segment_B: { cv: 35951, impression: 24312, ... },
  ...
}
  4. 計算各 event 差異值： 
    { cv: 4, impression: 3, ... }
  5. 用 event_key 順序尋找最小的 segment，將差異值加回該 segment。
  6. 完成差異補回後的 segment 結果即為對外提供的明細。

補充資料

  • DV360 變形小金剛數據
  • 受眾鎖定放大/縮小時的行為
  • 興趣議題透過 segment 打包
