1. [SuperDSP 平台化 Phase 1.3.0] IAS Pre-bid + 廣告實名制

原始頁面： https://guoshi.atlassian.net/wiki/spaces/ITO/pages/3779690498/SuperDSP+Phase+1.3.0+IAS+Pre-bid
最後修改：2025-04-21

緣由

搶攻廣告代理商的市場, 要推出可以被接受的 DSP 平台來搶食代理商這塊大餅

目標

  • 素材管理平台轉移至 Studio 中心化管理
  • SuperDSP 素材管理介面修改
  • Placement / Campaign 設定 BrandSafety
  • 因應代理商新增角色權限
  • 各角色於功能權責劃分
  • 廣告實名制要導入流程
  • 數據策展 介面皮
  • 流量主題策展

UserStory

  • Scrum 模式進行：第一批次： FE-Ryan, BE-Rocco；第二批次： FE-Ryan, Steve, BE-Rocco, Kirt, Elain

功能摘要

  • IAS Pre-bid：I/O單 placement level 新增/編輯支援 IAS 七大安全鎖定，複製 placement 功能需把所有 IAS 選項連帶複製過去，於 RichMedia / InStream / QualityTraffic 皆於「時段(小時)」上方，新增 BrandSafety 方案選擇與 IAS-BrandSafety 標題，頁面可收合。
  • 廣告實名制：設定 > 廣告主管理 > 新增/編輯廣告主，增加「實名制廣告主名稱」「公司統一編號」「公司地址」三個必填欄位，hover info 顯示說明，儲存時彈窗確認。
  • 廣告實名制：整體介面 Header 異動，廣告主下拉選單右邊加入 info，hover 顯示實名制資訊。
  • 廣告實名制：Sequence 底層邏輯，advertiser_name 來自簽約公司，payer_name 來自 SuperDSP 廣告主資料。
  • 廣告受眾鎖定功能、權限功能、Margin 計算等。

2. IAS-Prebid 及 廣告實名制

原始頁面： https://guoshi.atlassian.net/wiki/spaces/ITO/pages/3796795827/IAS-Prebid
最後修改：2025-02-18

Overview

  • 依據政府『詐欺犯罪危害防制條例』，需提供「受益人(委刊者)」與「出資者」資訊，新增「廣告主實名制」設定。
  • 代理商需要 IAS 七大安全鎖定功能。

目標

  • 新增「廣告主實名制」功能
  • 新增「IAS-Prebid」功能

目標使用者

  • External User（Client / Agency / Media，Admin / User）

預期效益

  • 符合法規要求，降低違規風險。
  • 提升代理商使用意願。

Change Scope

  • 廣告主管理：建立/編輯時新增「出資者名稱」欄位（必填，hover info 說明，blur 自動帶入，儲存彈窗確認）。
  • 廣告主管理：列表資料新增「出資者名稱」欄位。
  • Topbar：廣告主名稱旁新增 info，hover 顯示實名制資訊與免責聲明。
  • Ad Group：新增 IAS BrandSafety 設定（非必填，預設不選，於時段上方，API 取得選項）。

API 文件與相關連結

  • Sequences.json 欄位規格
  • 受眾管理
