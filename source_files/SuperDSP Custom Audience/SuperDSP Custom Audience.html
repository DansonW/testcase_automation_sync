SuperDSP 廣告組合 ad_group 修改 to_status api 邏輯判斷
ITP-2802 		Elain Wu
  • POST api/v1/superdsp/ad_groups/{id}/to_status
  • 狀態判斷時機：
    • 草稿 → 上刊
    • 暫停 → 上刊
    • 上刊中的 ad_group 進行更新
    • 每日 05:00 superdsp_ad_group_status（此次新增 00:00 的排程）
  • 狀態邏輯： 
    單一受眾包：
- 只有客製受眾包 & 打包完成 & 全部過期 => Draft → Paused
- 只有客製受眾包 & 打包完成 & 至少一個未過期 => Draft → On Cue
- 只有廣告/網站受眾包 & no ready => Draft → Reviewing
- 只有廣告/網站受眾包 & 至少一個 ready => Draft → On Cue

多種受眾包：
- 所有客製受眾包 & 打包完成 & 全部過期 + 任一廣告/網站受眾包 ready => Draft → On Cue
- 任一客製受眾包 & 打包完成 & 未過期 + 所有廣告/網站受眾包 no ready => Draft → On Cue
- 所有客製受眾包 & 打包完成 & 全部過期 + 所有廣告/網站受眾包 no ready => Draft → Reviewing
  • 審核中（Reviewing）優先權高於暫停，需先判斷：
    • 素材包 → 廣告/網站受眾包 → 客製受眾包
  • 狀態為審核中/暫停，需在 ad_group notes 記錄原因：
    • 素材審核未通過：review_pack_error
    • 廣告/流量受眾包未通過：review_audience_error
    • 客製受眾包未通過：review_custom_audience_error
  • 12/4 修改 oncue 判斷：改成判斷 include 的受眾包至少要過一包

3.4 Ad Group 儲存時新增 Custom Audience 過期狀態驗證

系統 SuperDSP
模組 Ad Group (廣告組合)
頁面 / 元件 Ad Group 新增/編輯抽屜
/DrawerAddAdGroup.vue
/CommerceAdDrawer.vue
開發人員 alice
預估時間 1.5D

功能說明

在 Ad Group 儲存時，新增驗證邏輯檢查數據鎖定中的 Custom Audience 狀態，若存在過期項目則提示使用者處理。

狀態驗證

  • 驗證時機：使用者點擊 Ad Group 儲存按鈕時
  • 驗證項目：Custom Audience 過期狀態
    • 條件：數據鎖定中的 "Custom Audience" 存在狀態為 Expired 的項目
    • 結果：彈出確認彈窗

確認彈窗

訊息：
此廣告組合包含已過期的受眾包，你可以更新受眾設定，或者在儲存廣告組合時，系統會自動將其從廣告組合中刪除。

按鈕 行為說明
仍要繼續 關閉確認彈窗；自動移除所有狀態為 Expired 的 Custom Audience（Include / Exclude）；繼續執行其他驗證 or 儲存流程
編輯受眾 關閉確認彈窗，將畫面捲動至「數據鎖定」區域
關閉按鈕 (X) 關閉彈窗，不執行任何動作

  • 抽屜標題：數據鎖定 - 客製受眾包
  • 抽屜組成：
    • 可選受眾包區域（左）：顯示可選擇的受眾包清單，提供篩選功能
    • 已選受眾包區域（右）：顯示當前已選取的受眾包項目，提供清除和取消選取功能
  • 受眾包項目資訊：
    • Audience Pack Name
    • Type 標籤

  • 參考元件：
    • DrawerTargetSegment.vue
    • DrawerAudiencePacks.vue
  • [已確認] 不會有廣告組合處於上刊狀態，受眾包狀態變更為 Processing 的情境

可選受眾區域

  • 顯示規則：
    • ✅ 顯示：Completed 狀態的受眾項目
    • ❌ 不顯示：Expired 項目（已過期項目會被過濾）
    • ⚠️ 例外：從右欄取消選取的 Expired 項目會回到此區域，但變更為停用狀態
  • 篩選功能：
    • 可依 Audience Pack Name、Type 篩選
    • 無符合條件時：顯示空白狀態 +「無資料」提示
  • 選擇行為：
    1. 選中項目後從左欄移除
    2. 出現在右欄「已選受眾」區域
    3. 檢查數量限制，達到上限時變更為 disabled 狀態

已選受眾區域

  • 數量顯示：根據已選項目數量即時變化
  • 移除方式：
    • 清除按鈕：清空所有已選項目
    • 單項取消：點擊已選項目 ⇒ 回到左欄「可選受眾」區域；Expired 項目回左欄後變為停用狀態

排序規則

按 ID 由新到舊（大→小），不受停用狀態影響。

狀態顯示

狀態 Icon Tooltip 訊息
Completed + UU = 0 ⓘ 無有效受眾
Expired ⚠ (紅色) 受眾包已過期，系統會在儲存廣告組合時自動將其從廣告組合中移除。

選取數量限制

區域 受眾包來源 限制數量
Include 特定族群、鑽石會員受眾、產業 Shopper Graph、Webb 零售數據、廣告活動受眾包、網站流量受眾包、客製受眾包 合計最多 100 個
Exclude 廣告活動受眾包、網站流量受眾包、客製受眾包 合計最多 50 個

超過限制時：可選區域選項狀態變更為 disabled，無法繼續選取。

儲存驗證

  • 點擊 Save 按鈕時，系統檢查已選受眾中是否包含 Expired 項目。
  • 有 Expired 項目時：顯示確認彈窗： 
    所選受眾包清單中包含已過期的受眾包。請點擊【取消】以更新受眾包設定，或點擊【確定】儲存設定，系統將自動移除已過期的受眾包。
    按鈕 行為說明
    OK 自動移除所有 Expired 項目後完成儲存，關閉抽屜
    Cancel 關閉彈窗，停留在抽屜頁面，可繼續調整選擇
    關閉按鈕 (X) 關閉彈窗，不執行任何動作
  • 無 Expired 項目時：直接儲存並關閉抽屜。

3.6 Ad Group 受眾包過期自動暫停機制

系統 SuperDSP
模組 Ad Group (廣告組合)
頁面 / 元件 Ad Group 列表 /ad-group/index.vue
開發人員 alice
預估時間 2D

基本資訊

  • 觸發條件：Ad Group 內的受眾包（Audience Pack）過期
  • 影響範圍：該 Ad Group 的投放狀態
  • 備註：
    • 只有當 Include 完全依賴 Custom Audience 且全部過期時才會自動暫停
    • 若 Include 中還有其他受眾類型，即使 Custom Audience 過期也不會暫停
    • 解決受眾包過期問題後，需要手動恢復上刊

自動暫停機制

  • 觸發條件（需同時滿足）：
    1. Ad Group 的數據鎖定 - Include 只有設定 Custom Audience
    2. 綁定的所有 Custom Audience 狀態都為 Expired
  • 觸發後行為：
    • 系統自動將該 Ad Group 的狀態變更為 Paused
    • 停止該 Ad Group 的廣告投放
  • 不觸發的情況：
    • Include 中還有其他受眾類型（特定族群、鎖右會員受眾、Shopper Graph、Webb 等）
    • 只有部分 Custom Audience 過期，仍有其他有效 Custom Audience
    • Custom Audience 僅設定在 Exclude

狀態顯示

  • Ad Group 列表 Status 欄位顯示「Paused」狀態標籤
  • 使用系統定義的 Paused 顏色
  • Tooltip：滑鼠 hover 時顯示： 
    此廣告組合選取的受眾皆已過期，廣告已暫停投放，請編輯受眾鎖定設定。

3.7 [Behavior Change] 廣告活動受眾包 / 網站流量受眾包 抽屜排序

系統 SuperDSP
模組 Ad Group (廣告組合)
頁面 / 元件 Ad Group 新增/編輯抽屜 > 數據鎖定 > 廣告活動受眾包 / 網站流量受眾包 抽屜
  • 變更範圍：廣告活動受眾包、網站流量受眾包兩個抽屜
  • 變更內容：調整項目排序邏輯，屬於行為變更（Behavior Change）
  • 排序規則：
    • 變更前：無明確排序
    • 變更後：依 ID 由新到舊（大 → 小）
    • 適用範圍：左欄可選受眾列表、右欄已選受眾列表
