移除 Campaign / Ad Group 花費欄位顯示限制

開發項目 / 需求 [列表與 DashBoard] 移除 Campaign/Ad Group 花費顯示限制
系統 SuperDSP
Ticket Number ITP-2877
開發人員 Ryan Meng
描述 目前系統對「花費 (Spend)」欄位設有上限邏輯（花費不可超過預算），現需移除此顯示限制，以呈現實際的廣告投放金額（含超跑狀況）。
影響範圍：廣告活動 (Campaign) 列表與 Dashboard、廣告組合 (Ad Group) 列表與 Dashboard。
異動核心：解鎖顯示限制、新增 (i) Hover 提示、API 結構重構（Summary 物件）。
備註 為配合後續列表日期連動需求，需同步重構 API 結構，由後端直接提供靜態的 Dashboard 統計資訊，確保全局加總數據不受前端動態篩選影響。

功能細項

  • 移除顯示限制：
    • 移除「花費上限等於預算」之邏輯，改為顯示實際投放金額（例如預算 $1,000，實際花費 $1,050 則顯示 $1,050）。
    • 此調整須同時套用於 廣告活動 與 廣告組合 之列表與 Dashboard。
  • 新增 Hover 提示：
    • 於「花費 / Spend」欄位標題旁新增 (i) 提示圖示。
    • zh：花費為實際廣告投放花費，帳務認列以預算設定為主。
    • en：Spend reflects the actual ad delivery amount. Billing is based on the budget setting.
  • API 結構重構：
    • 靜態 Dashboard 資訊：列表 API 需新增 dashboard 用的屬性，存放不隨列表分頁或動態過濾影響的總額數據。
    • 數據解耦：前端 Dashboard 元件須改為綁定此 API 靜態物件，不再透過前端列表加總計算。

Campaign / Ad Group 表格資訊呈現邏輯調整

開發項目 / 需求 [列表優化] 廣告活動與廣告組合列表：支援日期範圍即時數據查詢與匯出邏輯調整
系統 SuperDSP
Ticket Number ITP-2879
開發人員 Ryan Meng
描述 目前「廣告活動 (Campaign) 列表」與「廣告組合 (Ad Group) 列表」的指標數據（如曝光、點擊、花費等）為前端靜態過濾。需調整為與後端連動：當使用者切換 Date Picker 區間時，前端需重新發送 API 請求獲取該時段內的即時數據，確保介面與下載報表呈現內容一致。

功能細項

  • Date Picker 控管：
    • 廣告活動列表：依據活動走期限制選取範圍；無日期資訊 (未建立任何廣告活動) 時禁用設定。
    • 廣告組合列表：僅能選取該組合所屬廣告活動內的日期，超出範圍得日期須禁用選項。
  • 過濾方式：
    • 搜尋 / 狀態：維持前端過濾（過濾當前列表資料）。
    • 日期範圍：改為重新請求後端 API，而非前端過濾。
  • 下載報表：
    • 下載報表內容需呈現與 UI 上選取的日期區間結果相同。

[媒體 API] 路徑更新並新增 ad_format_type_ids 參數

開發項目 / 需求 [媒體 API] 路徑更新並新增 ad_format_type_ids 參數
系統 SuperDSP
Ticket Number ITP-2880
開發人員 Ryan Meng
描述 為配合後端架構遷移，本次預計將「媒體選項 (Media Options)」API 導向新的獨立後端專案。
主要變動：僅調整 /ad_groups/options/media 這支 API。
其餘 API：維持使用舊實例 requestV1 呼叫，不在此次改動範圍。
技術重點：前端透過 Vite Proxy 轉發至新專案網域，並新增 ad_format_type_ids 篩選參數。
備註 因首次在此系統加入新的後端 API 專案，後續測試需要請 mark 在機器上的 nginx config 加上以下設定：
location /api/v2/superdsp/ {
      proxy_pass https://sit-1-superdsp-api.onead.tw/;
      proxy_ssl_server_name on;
  }

功能細項

  • 新增請求參數：
    • API Request 新增 ad_format_type_ids 參數。
    • type：number[]
    • ad_format_type_ids為品類格式清單與廣告格式類型清單交集的 ad_format_type_ids
  • Vite Proxy 設定：
    • 於 vite.config 新增 /api/v2/superdsp 代理。
    • 設定 target 指向新 API 專案網域。(localhost:4002)
    • 啟用 rewrite 移除路徑前綴，確保路徑正確對接。
  • Axios 實例建立：
    • 於 axios 設定檔新增 requestV2 實例。
    • baseURL 設為 /api/v2/superdsp。
  • API 路徑更新：
    • 將原有的媒體 API 舊路徑更新為新版路徑 /ad_groups/options/media
    • 使用 requestV2 呼叫

受眾洞察報告放寬匯出資料日期條件

開發項目 / 需求 [受眾洞察報告] 開放當日數據匯出並放寬日期組件禁用條件
系統 SuperDSP
Ticket Number ITP-2882
開發人員 Ryan Meng
描述 原先因後端資料處理限制，限制廣告活動開始「後一天」才可匯出受眾洞察報告。現在問題已修正，需調整前端控管邏輯，開放活動開始「當天」即可進行報表查詢與匯出，並移除相關過時提示。

[報表] 修復數值加總格式並調整匯出欄位與 Sheet 結構

開發項目 / 需求 [Ad Group 報表] 修復數值加總格式並調整匯出欄位與 Sheet 結構
系統 SuperDSP
Ticket Number ITP-2881
開發人員 Ryan Meng
描述 目前廣告組合列表的下載報表，數值欄位僅以純文字或通用格式匯出，導致使用者在 Excel 中無法直接進行自動加總運算。此外，為了提升報表資訊的精準度，需同步移除冗餘欄位並優化 Sheet 呈現結構。

功能細項

  • 數值格式校正：
    • 目標：確保所有金額、百分比及數量欄位在 Excel 中可被辨識為「數值」格式。
    • 技術要求：使用 exceljs 的 numFmt 功能進行設定（例如：#,##0 或 0.00），解決目前下載後無法自動加總的問題。
  • 欄位異動：
    • 移除以下無效或不適用的欄位：
      • pricing_unit (計價單位)
      • average_price (含 RichMedia / Instream / Quality Traffic 單價、Commerce Ad 平均 CPC)
      • average_freq (平均頻次)
  • Sheet 結構與內容優化：
    • 首列資訊調整：
      • 第一列：顯示「搜尋區間」。
      • 第二列：
        • Total Sheet：顯示 Campaign name。
        • Ad Group Sheet：顯示 Ad Group Name。
    • Sheet 命名規範：
      • 第一張保持為 Total。
      • 其餘廣告組合分頁改為 Ad Group + ${id}（例如：Ad Group 12345）。

[L10n] 修正繁體中文介面「姓氏」與「名字」欄位標籤顯示顛倒問題

開發項目 / 需求 [L10n] 修正繁體中文介面「姓氏」與「名字」欄位標籤顯示顛倒問題
系統 SuperDSP
開發人員 Ryan Meng
描述 目前 SuperDSP 系統在「繁體中文」語系下，姓氏與名字的標籤 (Label) 顯示與實際資料內容完全顛倒。這不僅不符合中文姓名閱讀習慣，也會導致管理者在建立或編輯使用者時填入錯誤的資訊。
錯誤現象：標籤顯示「姓氏」的地方實際顯示的是 First Name；標籤顯示「名字」的地方實際顯示的是 Last Name。
受影響範圍：
  1. 側邊選單 - 我的帳號：個人資訊預覽區。
  2. 使用者管理列表：Table 欄位標頭與內容。
  3. 編輯/新增使用者彈窗：輸入框標籤。

功能細項

  • i18n 標籤校正：修正語系檔中 lastName 與 firstName 對應的中文翻譯，確保「姓氏」標籤對應到姓，「名字」標籤對應到名。
  • 校正確認：
    • 我的帳號：個人資訊預覽區顯示正確。
    • 使用者管理列表：Table 標頭與資料列對齊（姓氏顯示 Last Name）。
    • 編輯/新增使用者彈窗：輸入框 Label 顯示正確。

[預算控制] 修正廣告群組停刊（ABORTED）後預算未釋放的問題

開發項目 / 需求 [預算控制] 修正廣告群組停刊（ABORTED）後預算未釋放的問題
系統 SuperDSP
Ticket Number ITP-2921
開發人員 Ryan Meng
描述 當廣告群組狀態變更為「停刊」（含 CLOSED 或 ABORTED）後，系統計算 Campaign 總預算時，仍以該廣告群組原始設定的「預算總額（Budget）」而非「實際花費（Spent）」計算。這導致未用完的剩餘額度被鎖死，無法釋放回給同一個 Campaign 下的其他廣告群組使用。
現狀：僅 CLOSED 狀態會釋放預算。
問題：當狀態為 ABORTED 時，系統仍佔用完整預算額度。

功能細項

  • 邏輯擴充：修正預算計算公式，除原有的 CLOSED 狀態外，需將 ABORTED 狀態納入「釋放剩餘預算」的觸發條件。
  • 額度釋放計算：當廣告群組符合上述狀態時，該群組佔用的 Campaign 預算應從「原始設定金額 (Budget)」調整為「實際已花費金額 (Budget Spent)」。
  • 可用餘額連動：確保 Campaign 的「可用額度」能即時反映因群組停刊而釋放的差額，供其他廣告群組分配使用。
  • 校驗同步：
    • 新增廣告群組時，預算限制檢查須採計釋放後的正確餘額。
    • 編輯廣告群組時，剩餘預算的判斷邏輯須與此修正後的公式一致。
  • 數據一致性：確保此邏輯變更不影響過往已結束（CLOSED）廣告群組的預算顯示與計算基準。
