# 第一部分：系統入門

## 第一章：系統簡介

### 1.1 系統概述

ODM-UI（OneAD 廣告營運管理系統）是一套專為廣告營運團隊設計的管理平台，用於管理廣告活動的完整生命週期，包括活動建立、投放設定、素材管理、成效追蹤與報表匯出等功能。

![系統首頁總覽](./images/系統首頁總覽.png)

### 1.2 功能模組總覽

系統包含以下主要功能模組：

| 模組名稱 | 功能說明 | 網址 |
|---------|---------|------|
| 品牌廣告主 | 管理廣告主資料與關聯活動 | https://odm.onead.com.tw/advertiser/list |
| 廣告活動 | 建立與管理廣告活動 | https://odm.onead.com.tw/campaign/list |
| 投放項目 | 設定各類型投放版位 | https://odm.onead.com.tw/placement/list |
| 廣告素材 | 管理素材包、影片、橫幅 | https://odm.onead.com.tw/creative/list |
| 排 Cue 系統 | 委刊單管理與排程 | https://odm.onead.com.tw/cue_system_index |
| 活動追蹤 | 追蹤委刊回簽狀態 | https://odm.onead.com.tw/ad_request_status_memo |
| 上刊通報 | 管理上刊通報紀錄 | https://odm.onead.com.tw/campaign_launch_list |
| 活動進度 | 監控活動執行狀態 | https://odm.onead.com.tw/campaign_progress |
| 報表 | 查看與匯出成效報表 | 各活動/投放詳情頁 |
| SuperDSP 審查 | 審核 DSP 素材 | https://odm.onead.com.tw/superdsp_review |

### 1.3 角色與權限

系統依據使用者角色提供不同的功能權限：

| 角色 | 代碼 | 權限範圍 |
|-----|------|---------|
| 廣告營運管理員 | `adops_manager` | 完整功能存取權限 |
| 高級廣告營運管理員 | `adops_senior_manager` | 完整功能存取權限 |
| OneAD 廣告營運 | `adops_onead` | 基本營運功能、CIS 成員功能 |
| OneAD GF 廣告營運 | `adops_onead_gf` | 基本營運功能 |
| GF 廣告營運 | `adops_gf` | 基本營運功能 |
| CPS 主任 | `cps_director` | CPS 相關功能 |
| CPS 銷售 | `cps_sales` | CPS 相關功能 |
| 設計管理員 | `art_manager` | 素材相關功能 |

#### 權限動作說明

系統權限採用 `動作:資源` 格式（例如 `create:campaign`），各動作定義如下：

| 動作 | 代碼 | 說明 |
|-----|------|------|
| 新增 | `create` | 建立新資源的權限 |
| 檢視 | `show` | 查看單一資源詳情的權限 |
| 列表 | `index` | 查看資源列表的權限 |
| 編輯 | `update` | 修改現有資源的權限 |
| 刪除 | `destroy` | 移除資源的權限 |
| 管理 | `manage` | 資源的完整管理權限（包含所有動作） |

#### 權限資源說明

| 資源 | 代碼 | 說明 |
|-----|------|------|
| 廣告活動 | `campaign` | 活動的建立、檢視、編輯權限 |
| 投放項目 | `placement` | 版位投放的建立、檢視、編輯權限 |
| 素材包 | `pack` | 素材包的建立、檢視、編輯、刪除權限 |
| 追蹤碼 | `tracking_code` | 追蹤碼的建立、編輯、刪除權限 |
| 排程系統 | `cue_list` | 排程列表的檢視權限 |
| 上刊列表 | `campaign_launch_list` | 活動上刊列表的檢視權限 |

#### 角色群組

系統將角色分為以下群組，用於特定功能的存取控制：

| 群組 | 包含角色 | 功能存取 |
|-----|---------|---------|
| 管理員 | `adops_manager`、`adops_senior_manager`、`cps_director` | 進階管理功能 |
| CIS 成員 | `adops_onead`、`adops_manager`、`adops_senior_manager` | 活動進度追蹤 |
| CPS 成員 | `adops_onead_gf`、`adops_gf`、`cps_director`、`cps_sales` | CPS 相關功能 |
| 設計管理員 | `art_manager` | 素材審核管理 |

> **重要提醒**
>
> 使用者的角色與權限設定由 **ERP 系統** 統一管理。如需新增、修改或調整權限，請聯繫：
> - 您的直屬主管提出申請
> - ITO 負責人員進行設定
>
> ODM 系統會在登入時自動從 ERP 取得您的權限設定。

### 1.4 核心工作流程

廣告活動的標準作業流程如下：

```
1. 建立/選擇廣告主
        ↓
2. 建立廣告活動（設定基本資訊、預算、走期）
        ↓
3. 建立投放項目（選擇版位類型、設定目標受眾）
        ↓
4. 綁定廣告素材（選擇或建立素材包）
        ↓
5. 提交審核 → 主管核准
        ↓
6. 活動上線執行
        ↓
7. 監控成效與報表追蹤
        ↓
8. 活動結案
```

## 第二章：登入與帳戶管理

### 2.1 系統登入

#### 操作步驟

1. 開啟瀏覽器，輸入系統網址
2. 在登入頁面輸入您的帳號與密碼
3. 點擊「登入」按鈕
4. 登入成功後，系統將自動導向首頁

![登入頁面](./images/登入頁面.png)

| 欄位名稱 | 必填 | 說明 | 格式限制 |
|---------|------|------|---------|
| 帳號 | 是 | 使用者電子郵件 | 有效的 Email 格式 |
| 密碼 | 是 | 使用者密碼 | 至少 8 個字元 |

> **注意事項**
> - 連續輸入錯誤密碼 5 次後，帳號將被暫時鎖定
> - 如忘記密碼，請聯繫系統管理員重置

### 2.2 帳戶設定

#### 查看帳戶資訊

1. 點擊頁面右上角的使用者圖示
2. 選擇「帳號設定」
3. 系統將顯示您的帳戶資訊頁面

![帳戶設定頁面](./images/帳戶設定頁面.png)

#### 編輯帳戶資訊

1. 在帳戶設定頁面，點擊「編輯」按鈕
2. 修改需要更新的欄位
3. 點擊「儲存」完成修改

| 欄位名稱 | 必填 | 說明 | 可否編輯 |
|---------|------|------|---------|
| 使用者名稱 | 是 | 顯示名稱 | 是 |
| 電子郵件 | 是 | 登入帳號 | 否 |
| 角色 | - | 使用者權限角色 | 否（由管理員設定）|

### 2.3 密碼變更

#### 操作步驟

1. 進入帳戶設定頁面
2. 點擊「變更密碼」選項
3. 輸入目前密碼
4. 輸入新密碼（兩次確認）
5. 點擊「確認變更」

![變更密碼介面](./images/變更密碼介面.png)

| 欄位名稱 | 必填 | 說明 | 格式限制 |
|---------|------|------|---------|
| 目前密碼 | 是 | 驗證身份用 | - |
| 新密碼 | 是 | 新設定的密碼 | 至少 8 個字元，需包含英數字 |
| 確認新密碼 | 是 | 再次輸入新密碼 | 需與新密碼一致 |

> **密碼安全建議**
> - 定期更換密碼（建議每 90 天）
> - 避免使用與其他網站相同的密碼
> - 密碼應包含大小寫英文、數字及特殊符號

## 第四章：廣告活動管理

#### 列表欄位說明

| 活動名稱 | 廣告活動名稱 |
| 廣告主 | 所屬品牌廣告主 |
| 業務窗口 | 負責業務人員 |
| 走期 | 活動開始與結束日期 |
| 狀態 | 活動目前狀態 |
| 預算 | 活動總預算金額 |

#### 搜尋功能

| 搜尋類型 | 說明 | 操作方式 |
|---------|------|---------|

| 活動 ID | 依活動 ID 精確搜尋 | 選擇「ID」後輸入編號 |

### 4.2 建立廣告活動

> **權限要求**：需具備 `create:campaign` 權限

#### 操作步驟

1. 先進入目標廣告主的詳情頁面
2. 點擊「新增活動」按鈕
3. 填寫活動基本資訊表單
4. 點擊「儲存」建立活動

![新增活動表單](./images/新增活動表單.png)

#### 基本資訊欄位

| 欄位名稱 | 必填 | 說明 | 格式限制 |
|---------|------|------|---------|
| 活動名稱 | 是 | 廣告活動名稱 | 最多 200 字元 |
| 活動走期 | 是 | 活動執行期間 | 開始日期不得早於今日 |
| 業務窗口 | 是 | 負責業務人員 | 從下拉選單選擇 |
| 代理商/媒體 | 是 | 選擇代理商或媒體 (Publisher Speed) | 依電台按鈕選擇 |
| 產業類別 | 是 | 廣告主產業類別與子類別 | 從下拉選單選擇 |
| OneAD產業類別 | 是 | OneAD 內部定義之產業類別與子類別 | 從下拉選單選擇 |
| 活動備註 | 否 | 補充說明 | 最多 1000 字元 |

#### 預算設定欄位

| 欄位名稱 | 必填 | 說明 | 格式限制 |
|---------|------|------|---------|
| 總預算 | 是 | 活動總預算金額 | 正整數，單位：元 |
| 贈送金額 | 否 | 額外贈送的預算 | 正整數，單位：元 |
| 幣別 | 是 | 預算幣別 | 預設為 TWD |
| 涵蓋第三方媒體 | 否 | 是否包含第三方媒體廣告 | 勾選後需填寫下方第三方預算資訊 |
| 第三方預算 | 視勾選 | 第三方媒體之預算 | 正整數 |
| 第三方贈送金額 | 視勾選 | 第三方媒體之贈送金額 | 正整數 |
| 接單公司 | 視勾選 | 處理第三方訂單之公司 (如：果然) | 若選擇「果然」，需額外填寫「折扣」 |

#### 操作步驟

1. 在活動列表中，點擊目標活動的「編輯」按鈕
2. 或在活動詳情頁點擊「編輯」

### 4.4 活動詳情查看

> **權限要求**：需具備 `show:campaign` 權限

#### 進入活動詳情

1. 在活動列表點擊活動名稱
2. 或點擊「查看」按鈕

![活動詳情頁面](./images/活動詳情頁面.png)

#### 詳情頁面區塊

| 區塊名稱 | 內容說明 |
|---------|---------|
| 基本資訊 | 活動名稱、走期、業務窗口、狀態等 |
| 預算摘要 | 總預算、已消耗、剩餘預算 |
| 投放項目列表 | 該活動下所有投放項目 |
| 成效摘要 | 曝光數、點擊數、CTR 等關鍵指標 |

### 4.5 主管核准流程 (Approval Workflow)

#### 第一階段：提交核准 (營運人員)
1. **進入活動詳情**：在「廣告活動列表」點擊目標活動名稱。
2. **啟動核准程序**：點擊頁面右上方的 **「提交核准」** 按鈕。
3. **最後確認**：在系統提示彈窗點擊「確認提交」。
   - 此時活動狀態將由「草稿 (Draft)」轉變為 **「待審核 (Pending)」**。
   - 系統會自動發送通知給具備審核權限的主管。

#### 第二階段：執行審核 (主管人員)
1. **檢視待審清單**：主管登入後，可透過狀態篩選功能找出所有「待審核」的活動。
2. **審閱設定內容**：點擊進入活動詳情，重點檢查：
   - 走期是否正確、預算金額是否超出授權、投放類別是否精確。
3. **做出審核判定**：
   - **核准 (Approve)**：點擊「核准」按鈕，活動狀態轉為 **「已核准 (Approved)」**，可準備上線。
   - **退回 (Reject/Return)**：
     1. 點擊「退回」按鈕。
     2. 在彈窗中輸入 **「退回原因」**。
     3. 點擊確認，活動狀態將回退至「草稿 (Draft)」供營運人員修改。

![主管核准介面](./images/主管核准介面.png)

### 4.6 手動結案

#### 適用情境

- 活動已完成目標，提前結束
- 客戶要求提前終止活動
- 其他需要手動結束活動的情況

#### 操作步驟

1. 進入活動詳情頁
2. 點擊「結案」按鈕
3. 選擇結案原因
4. 填寫結案備註（選填）
5. 確認結案

![活動結案介面](./images/活動結案介面.png)

> **注意事項**
> - 結案後，所有關聯的投放項目將停止投放
> - 結案操作無法復原，請謹慎操作

## 第五章：投放項目設定

### 5.1 版位類型總覽

系統支援五種投放版位類型：

| 類型 | 代碼 | 說明 | 特點 |
|-----|------|------|------|
| Speed | `speed` | 速購投放 | 快速觸及、保證曝光 |
| RON | `RON` | 全網投放 | Run of Network，全網分配 |
| DSP | `DSP` | 程序化投放 | Demand-Side Platform 程序化購買 |
| External | `external` | 外部投放 | 第三方平台追蹤 |
| Publisher Speed | `publisher_speed` | 媒體速購 | 特定媒體保證量 |

### 5.2 進入投放項目列表

1. 從左側選單點擊「投放項目」
2. 系統顯示所有投放項目列表

### 5.8 投放狀態流轉

投放項目的狀態會隨著生命週期變化：

```
草稿 (Draft)
    ↓ [提交]
待發佈 (Pending)
    ↓ [核准/自動發佈]
進行中 (Active)
    ↓ [暫停]     ↓ [結束日期到達]
已暫停 (Paused)  已完成 (Completed)
    ↓ [恢復]         ↓ [封存]
進行中 (Active)  已封存 (Archived)
```

| 狀態 | 說明 | 可執行操作 |
|-----|------|-----------|
| 草稿 | 尚未提交的投放 | 編輯、提交、刪除 |
| 待發佈 | 等待核准或排程中 | 編輯、取消 |
| 進行中 | 正在投放中 | 暫停、編輯部分欄位 |
| 已暫停 | 暫時停止投放 | 恢復、編輯 |
| 已完成 | 投放結束 | 查看報表、封存 |
| 已封存 | 已歸檔的投放 | 查看報表 |

#### 操作步驟

1. 在 Cue 表列表頁面，點擊「新增 Cue 表」

3. 設定投放配置
4. 點擊「儲存」建立 Cue 表

![新增 Cue 表表單](./images/新增_Cue_表表單.png)

#### 基本資訊欄位

| 欄位名稱 | 必填 | 說明 |
|---------|------|------|
| Cue 名稱 | 是 | Cue 表名稱 |
| 廣告主 | 是 | 選擇廣告主 |
| 業務窗口 | 是 | 負責業務人員 |
| 代理商 | 否 | 媒體代理商 |
| 預計走期 | 是 | 預計執行日期範圍 |
| 總預算 | 是 | 預算金額 |
| 贈送金額 | 否 | 額外贈送預算 |

#### 投放配置與產品線設定

在 Cue 表中可預先規劃多筆產品線 (Product Line) 的投放配置，包含 OneAD 與第三方(External)產品。

**新增產品線**
1. 點擊「新增產品」，選擇「產品線」並設定「買法」(Purchase Way)。

**設定 OneAD 廣告格式與預算**
1. 點選「新增產品」以加入 OneAD 廣告格式。
2. 選擇「版位」、「影片秒數」並輸入「預估頻次上限」。
3. 點選「新增廣告預算」，於彈窗中輸入「廣告預算」、「走期」、「計價單位」(如 CPM) 以及「單價」。
4. 點擊確認。

**設定第三方 (External) 廣告格式與預算**
1. 點選「新增產品」以加入第三方廣告。
2. 依序選擇「媒體」、「格式」、「計價單位」、「版位」及「經銷商」。
3. 點選「新增廣告預算」，輸入「廣告預算」、「服務費」、「毛利率」、「走期」、「裝置」、「單價」與「指標」。
4. 點擊確認。

**儲存 Cue 表**
- 完成所有設定後，點擊「儲存」按鈕。系統會彈出「確定要儲存嗎？」的確認視窗，點擊確認即可建立。

## 第十三章：SuperDSP 素材審查

> **權限要求**：此功能僅限管理員存取

### 13.1 功能概述

SuperDSP 素材審查用於審核從 DSP 平台提交的廣告素材。

![SuperDSP 審查頁面](./images/SuperDSP_審查頁面.png)

### 13.2 進入審查頁面

1. 從左側選單點擊「SuperDSP 審查」
2. 系統驗證管理員權限
3. 顯示待審核素材列表

### 13.3 待審核列表

#### 列表欄位說明

| 欄位名稱 | 說明 |
|---------|------|
| 素材 ID | 素材唯一識別碼 |
| 廣告主 | 所屬廣告主 |
| 素材類型 | 影片/橫幅 |
| 提交時間 | 素材提交審核時間 |
| 狀態 | 待審核/已核准/已拒絕 |

### 13.4 審核流程

### 13.4 審核操作流程

#### A. 進行新素材審核 (Reviewing)
1. **進入審查頁面**：從選單選擇「SuperDSP 審查」，系統預設進入 **「Reviewing (審核中)」** 頁籤。
2. **選取待審素材**：在列表中找到欲處理的素材 ID。
3. **預覽與檢查**：點擊素材 ID 開啟預覽視窗，檢查視覺內容、廣告標題、轉址連結是否符合平台規範。
4. **執行審核判定**：
   - **核准通過 (Pass)**：
     1. 點擊素材旁的 **「Pass」** 按鈕。
     2. 系統彈出類別設定視窗，從下拉選單選擇正確的 **「產業類別 (Category)」** 與 **「子類別 (Sub Category)」**。
     3. 點擊 **「Confirm」** 按鈕完成，素材將移至「Passed」頁籤。
   - **拒絕核准 (Reject)**：
     1. 點擊素材旁的 **「Reject」** 按鈕。
     2. 在彈出的輸入框內填寫具體的 **「拒絕原因 (Reject Reason)」**。
     3. 點擊 **「Confirm Reject」** 按鈕，素材將移至「Rejected」頁籤。

#### B. 調整已審核結果
* **撤銷核准 (退回重新審核)**：
  1. 切換至 **「Passed」** 頁籤。
  2. 找到目標素材，點擊 **「Back to Reviewing」** 按鈕。
  3. 素材狀態將退回「Reviewing」，可重新執行核准或拒絕動作。
* **將拒絕件轉為通過**：
  1. 切換至 **「Rejected」** 頁籤。
  2. 找到目標素材，點擊 **「Pass」** 按鈕。
  3. 依據上述核准步驟重新設定產業類別並確認。

![素材審核介面](./images/素材審核介面.png)

#### 審核標準

素材審核需確認：
- 素材內容符合廣告政策
- 素材品質達到標準
- 沒有違規或不當內容
- 素材規格符合要求

### 13.5 拒絕原因

常見的拒絕原因包括：

| 原因分類 | 說明 |
|---------|------|
| 內容不當 | 含有違規、不雅內容 |
| 品質不佳 | 解析度低、模糊 |
| 規格不符 | 尺寸、格式不符要求 |
| 品牌侵權 | 未授權使用品牌內容 |
| 誇大不實 | 廣告內容有誇大嫌疑 |

### 13.6 審核歷史

查看素材的審核歷史紀錄。

1. 點擊素材進入詳情
2. 切換至「審核歷史」標籤
3. 顯示所有審核紀錄

## 附錄 B：常見問題（FAQ）

### B.1 登入與帳戶

**Q1：忘記密碼怎麼辦？**

A：請聯繫系統管理員重置密碼。管理員會發送新的臨時密碼至您的註冊信箱。

**Q2：帳號被鎖定怎麼處理？**

A：連續輸入錯誤密碼 5 次後帳號會被暫時鎖定，請等待 30 分鐘後重試，或聯繫管理員解鎖。

**Q3：如何變更我的顯示名稱？**

A：進入「帳號設定」→「編輯」，修改使用者名稱後儲存即可。

---

### B.2 廣告活動

**Q4：活動建立後可以修改走期嗎？**

A：可以。在活動尚未開始執行前可自由修改。活動進行中時，只能延長結束日期，無法提前開始日期。

**Q5：為什麼我無法建立新活動？**

A：請確認您具備 `create:campaign` 權限。如沒有此權限，請聯繫管理員授權。

**Q6：活動結案後可以恢復嗎？**

A：不行，活動結案後無法恢復。如需重新投放，請建立新的活動。

---

### B.3 投放項目

**Q7：投放上線後可以修改設定嗎？**

A：投放進行中時，部分欄位可以修改（如預算、Pacing），但基本設定（如投放類型）無法變更。

**Q8：為什麼投放沒有開始投放？**

A：請檢查以下項目：
1. 投放走期是否已開始
2. 投放狀態是否為「進行中」
3. 是否有綁定素材
4. 活動狀態是否為「進行中」

**Q9：如何暫停投放？**

A：在投放詳情頁點擊「暫停」按鈕，投放會立即停止。恢復時點擊「恢復投放」。

---

### B.4 素材管理

**Q10：支援哪些影片格式？**

A：系統支援 MP4 和 MOV 格式，建議使用 H.264 編碼，檔案大小上限 500MB。

**Q11：橫幅圖片有尺寸限制嗎？**

A：有。請依據選擇的廣告格式上傳對應尺寸的圖片，常見尺寸包括 300x250、728x90、320x50、300x600。

**Q12：素材上傳後多久可以使用？**

A：圖片素材上傳後即可使用。影片素材需等待轉檔完成，通常需要幾分鐘。

---

### B.5 報表

**Q13：報表數據多久更新一次？**

A：報表數據每小時更新一次。如需最即時的數據，請稍後再查看。

**Q14：匯出的 Excel 報表亂碼怎麼辦？**

A：匯出的檔案為 UTF-8 編碼，請使用 Excel 的「資料」→「從文字檔」功能匯入，並選擇 UTF-8 編碼。

**Q15：為什麼報表數據與第三方工具不一致？**

A：不同平台的計算方式可能略有差異（如曝光認定標準）。建議以 ODM-UI 報表為主要參考。

---

### B.6 系統操作

**Q16：頁面載入很慢怎麼辦？**

A：請嘗試以下方式：
1. 重新整理頁面
2. 清除瀏覽器快取
3. 檢查網路連線
4. 使用 Chrome 或 Firefox 瀏覽器

**Q17：操作後沒有反應怎麼處理？**

A：請檢查：
1. 是否有必填欄位未填
2. 欄位格式是否正確
3. 網路連線是否正常
4. 頁面是否需要重新整理

**Q18：批次更新失敗怎麼辦？**

A：系統會顯示失敗的項目與原因，請根據錯誤訊息修正後重試。常見原因包括投放狀態不允許修改、數值超出範圍等。
