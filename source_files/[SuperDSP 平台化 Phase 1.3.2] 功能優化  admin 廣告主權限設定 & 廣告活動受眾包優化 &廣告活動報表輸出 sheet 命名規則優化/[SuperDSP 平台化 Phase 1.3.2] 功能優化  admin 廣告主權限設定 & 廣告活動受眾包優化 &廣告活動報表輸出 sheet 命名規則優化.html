📚 用戶故事與用例

標題 角色 需求 目的 優先級 驗收標準
新增廣告主時，admin 的廣告主權限需自動被加入 Agency_Admin / Client_Admin / Media_Admin 在新增「廣告主」時，自動取得該廣告主的所有查看存取權限 以便能即時查看與管理新建立之廣告主的所有資訊，無需額外申請權限 高 1. 任一合約(Agency/Client/Media)下的任意帳號建立新「廣告主」後
2. Admin 帳號（包含 agency_admin, client_admin, media_admin）重整頁面
3. Admin 帳號應自動取得新建立之「廣告主」的所有查看存取權限
4. 若廣告主停用，權限變更應立即生效，無需人工干預
廣告活動受眾打包，增加 Impression 事件（看過廣告） SuperDSP 的 User（具有受眾管理權限的用戶） 我需要在受眾管理 > 廣告活動受眾包中增加「看過廣告」的打包選項 以便能更精確地鎖定特定受眾群體，優化廣告投放效果 高 1. 在「受眾管理 > 廣告活動受眾包」介面中，新增「看過廣告」的打包條件選項
2. 用戶可以成功選擇「看過廣告」條件進行受眾打包
3. 用戶可以完成從建立 → 打包 → 被選擇 → 投放的整套流程
4. 打包後的受眾資料能被正確儲存並在廣告投放時被調用
5. 確保打包功能與其他現有條件選項相容，不產生衝突

4. [SuperDSP 優化需求] 廣告活動報表輸出 sheet 命名規格優化

原始頁面：https://guoshi.atlassian.net/wiki/spaces/ITO/pages/3869048833/SuperDSP+sheet
最後修改：2025-05-23

現況：

  • 使用者會將 廣告組合名稱 命名超長，而且重複性很高如下圖，造成輸出 excel 報表的時候會產生兩個狀況：
  • 1. 名稱超過一定長度時, 會截取至固定字元做輸出
  • 2. 以下命名情境下, 三個 廣告組合名稱 截取出來的名稱結果都是一樣的, 因次造成輸出名稱之結果重複命名並非 uniqe , 所以下載報表的錯誤因此產生。
