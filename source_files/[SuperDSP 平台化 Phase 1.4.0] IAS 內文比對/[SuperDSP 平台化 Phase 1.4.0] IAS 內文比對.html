📖 項目概述

📌 項目背景

在前一次 品牌安全 功能的基礎上，新增 內文比對 以豐富 pre-bid 廣告投放監控功能。

🎯 項目目標

  • 數據鎖定增加「Environment」模式選項，保留舊有「Audience」。
  • 「Audience」及「Environment」兩者擇一設定，不可同時啟用。
    • Audience：年齡、興趣特定議題、包含鎖定、排除鎖定
    • Environment：包含內文比對、排除內文比對
    • 兩者一起設定會高機率面臨廣告投放不完風險（ODM 雖可同時設定，但實務會切開 Placement）。
    • 上刊後，Audience / Environment 不可互相切換（技術可行，但報表難解讀）。

    切換情境與數據說明：

    廣告活動先用 Audience 執行
數據收：segment 100 / 年齡 100

切換 Environment 執行
數據收：網站 100 / 年齡 100 + 年齡 100

必須三個維度同時輸出，才看得出 segment 100 / 網站 100 / 年齡 200 的關係
  • Placement Sequence 新增「內文比對」 key contextual（參考 OSS Spec：https://guoshi.atlassian.net/wiki/spaces/OSS/pages/8781860）。
  • ERP 設定介面 CUD 後，可同步更新 SuperDSP 與 ODM 的「品牌安全」及「內文比對」選項（資料同源）。

🧩 項目範圍

包含：

  • 數據鎖定新增「Audience」與「Environment」兩分類：
    • 兩者擇一設定，不能同時啟用。
    • Audience：選項可正常 CRUD，且量體預估與報表輸出皆正常。
    • Environment：包含/排除內文比對可正常 CRUD。
  • 內文比對 Placement Sequence for SuperDSP 邏輯調整：
    • key contextual 加入，並可與 ad_server 正常溝通。
  • ERP 設定介面可即時且同步更新 SuperDSP 與 ODM 兩系統的「品牌安全」與「內文比對」選項。

不包含：

  • 品類對於內文比對之差異：
    • InStream 不開放內文比對
    • RichMedia, Quality Traffic 開放內文比對
  • 合約對於內文比對之差異：
    • Media 合約不開放內文比對
    • Agency, Client 合約開放內文比對
  • 內文比對 Placement Sequence for ODM 邏輯調整（2024.12 已實現）。

[SuperDSP 功能需求] 廣告組合 之 受眾鎖定 新增項目分類

原始文件連結： https://guoshi.atlassian.net/wiki/spaces/ITO/pages/3859677213/SuperDSP

隨著受眾鎖定策略的演進，擴展了分類維度，增加了「Audience」與「Environment」兩大類別：
「Audience」：以人為本的定位策略。
「Environment」：以環境為本的投放邏輯。

介面規格

  • 兩模式介面表現請參考 Figma： Figma UI/UX
  • 若已設定 A 類別，切換到 B 類別時會跳出提示： 
    您即將切換至 B 數據鎖定設定。
目前已設定的 A 內容將被清除且無法復原。
系統不支援廣告同時鎖定 A 與 B。 
請確認是否要繼續切換？
[取消] [確定]

（Audience 與 Environment 互切時的四種提示文案，同理）

[SuperDSP 功能需求] 受眾鎖定 之「Environment」 新增內文比對項目

原始文件連結： https://guoshi.atlassian.net/wiki/spaces/ITO/pages/3860135937/SuperDSP+Environment

前端

  • Environment 的內文比對項目請參考 Figma： Figma UI/UX
  • 介面基本同「廣告活動受眾」設定，右側多一個 filter icon 提供內文比對分類下拉清單：
    • 下拉清單項目由後端提供；預設為「所有類別」。
    • 原有 CCT_ / CCA_ prefix 會被移除（由 seed 改名）。
    • 本階段 1.4.0 沿用現有分類，下一階段才會有 SuperDSP 專屬分類系統。
  • 切換到 Environment 時，畫面移除「量體預估」功能（因無環境數據可估）。

後端

  • Environment 之內文比對全 CRUD。
  • 當選擇 Environment 時，Sequence 會將所有性別年齡全選後輸出。
  • 情境卡控：
    • 廣告組合 Create/Update 若鎖定內容為空，會跳出提示；按「確定」後，再進入編輯會看到受眾模式且年齡性別全選。

查看權限

角色 說明
SuperDSP - Agency_User / Client_User
  • RichMedia, Quality Traffic 品類：
    • 有查看權限：可查看廣告組合的內文比對與項目。
    • 有編輯權限：可建立/編輯廣告組合內文比對。
  • InStream：維持舊畫面。
SuperDSP - Media_User 維持舊畫面。

[SuperDSP 優化需求] 近期開發功能 - 介面導入英文版

原始文件連結： https://guoshi.atlassian.net/wiki/spaces/ITO/pages/3855220737/SuperDSP+-

動機：因應公司國際化，將介面補齊中英文。

現況：

  • 廣告組合「品牌安全」介面沒有英文版。
  • 廣告組合「數據鎖定」介面沒有英文版。

期望：

  1. 品牌安全：標題與內容選項導入英文版（詳見 Google Slides 規格）。

  3. SuperDSP 轉跳 Studio 的 Popup 訊息導入英文版。

  • 廣告組合狀態異動時，最大起訖日可能跟著變化。
