[Partners][SuperDSP 平台化 Phase 1.5.0] Commerce Ad

原始文件連結： https://guoshi.atlassian.net/wiki/spaces/ITO/pages/3928522753/Partners+SuperDSP+Phase+1.5.0+Commerce+Ad

緣由

SuperDSP 推出 新格式 Commerce Ad，Partners 也會同步增加，因此需要通知使用者 Partners 有新格式可以使用。
而 Commerce Ad 在公司內部歸類在 NativeDrive 原生廣告 底下的廣告單元（格式）。

目標

  • 對使用者顯示新功能可以使用的資訊彈窗，所有使用者至少會看到通知 1 次
  • Partners / 廣告設定 / NativeDrive 原生廣告 中增加 Commerce Ad 手機 / 電腦版的版位預覽畫面
  • 功能上線後，已申請過 NativeDrive 原生廣告 的使用者，Commerce Ad 的合約會先建立完成
  • 上線後，未申請過 NativeDrive 原生廣告 的使用者，按下「儲存並取得程式碼」後，將會在資料庫建立 NativeDrive 原生廣告與 Commerce Ad 的合約資料

功能細項

系統 User Story ID 前/後端 功能 細項 Who 開發時間 描述
Partners 1 前 通知彈窗
  • 顯示 Commerce Ad 格式新增的資訊
  • 彈窗中需要有「下次不再顯示」的核選方框供使用者勾選
  • 彈窗底部需要有「關閉」按鈕，右上也有 X 供使用者關閉彈窗
  • 資訊文字需與 PAD 索取
  • 運用前端 Local Storage 紀錄使用者是否勾選「下次不再顯示」
  • 彈窗顯示只需 2 個月即可，若 8 月底上，可設定 10 月底不再顯示彈窗的判斷條件
  • 加入備註說明 彈窗程式碼可在何時以後移除
Partners 2 前 廣告設定 / NativeDrive 原生廣告的廣告格式預覽
  • 原本的 NativeDrive 原生廣告 手機 / 電腦版預覽
  • Commerce Ad 電腦版預覽
  • 預覽的樣式需要與 MTO 索取
Partners 3 後 更新舊合約資料
  • 已申請過 NativeDrive 原生廣告的使用者應增加 Commerce Ad 合約
  • 針對已申請過 NativeDrive 原生廣告的使用者
  • 需建立資料表 media_給後端決定_contract_terms，並用 seed 來建立 Commerce Ad 合約資料
  • Commerce Ad 合約資料的 price_info 為：
{
  onead_price: 2,
  media_price: 2,
  gsp_buy_tech_license_fee: 0.5,
  media_buy_tech_license_fee: 0,
  rebate: 0
}
Partners 4 後 按下「儲存並取得程式碼」後呼叫的 API
  • 確保建立合約的後端邏輯已經包含新合約，總共會建立 NativeDrive & Commerce Ad 2 種合約
  • 針對上線後未申請過 NativeDrive 原生廣告的使用者
