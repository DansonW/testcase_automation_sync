📖 項目概述

📌 項目背景

SuperDSP 過往所有素材建立流程皆在平台內處理，導致因應新格式需求時，前端、後端、MTO 皆需反覆開發與調整，增加了跨部門 loading 與維護風險。 為了提升擴展彈性並降低內部負擔，將導入 Studio 系統作為主要素材管理平台。

🎯 項目目標

  • 將素材建立流程導流至 Studio，支援快速上線新廣告格式
  • 在 SuperDSP 的素材管理頁新增「建立更多素材」按鈕
  • 導入跳轉至 Studio 建立素材包的流程（帶入 user token）
  • 將追蹤碼設定邏輯由素材包階段改為 Placement 階段
  • 根據合約格式自動判斷格式來源（Studio / 系統原生），進行功能切換

🧩 項目範圍

包含：

  • SuperDSP UI 增設「前往 Studio 建立素材」按鈕並導向 Studio（名稱暫定）
  • 跳轉到 OneAD Studio 時，需新開畫面後透過後端 redirect 到 OneAD Studio
  • 修改 Placement 建立流程支援設定追蹤碼
  • 格式識別邏輯與 UI 行為（Studio 格式需新開視窗導至對應格式建立畫面）
  • Studio 系統內部素材建立流程，移出設定追蹤碼功能

不包含：

  • 原生 SuperDSP 格式的 UI / 欄位邏輯變動

🎯 使用場景說明

主要包含四種使用情境：

  1. 從素材管理頁點擊「前往 Studio 建立素材」導向行為
  2. 使用者欲建立素材包時，選擇 Studio 格式並導向 Studio 建立頁
  3. 素材管理頁列表中的 Studio 欄位不可編輯，只能查看已使用項目
  4. 建立廣告組合時選擇 Studio 格式，禁用素材包建立按鈕並提示

並對應 token 失效、使用者取消跳轉等異常處理。

Banners / Videos / Packs

文件中詳細列出：

  • 撈取 banners：/api/v1/superdsp/companies/#{client_id}/banners，回傳 banner_source（SuperDSP / OneAD Studio）
  • 撈取 videos：/api/v1/superdsp/companies/#{client_id}/videos，回傳 video_source
  • 撈取 packs：/api/v1/superdsp/companies/#{client_id}/packs，回傳 pack_source
  • 建立 / 更新 pack 時移除 tracking_codes 欄位，改由 AdGroup 維護
  • 查詢單一 pack 時 response 中移除 tracking_codes

AdGroup

文件定義：

  • 撈取 Ad Group（含 instream）：GET /api/v1/superdsp/ad_groups(/instream)/{ad_group_id}，回傳 tracking_codes
  • 建立 / 更新 Ad Group（含 instream）時，body 加入 tracking_codes: String Array
  • 在 UseCases::SuperdspAdGroupCreateUseCase#tracking_code 中將每筆 tracking_code 轉為 intent/custom/pixel/event_type_id=1 的結構

MASA - API 規格

Users - refresh_token

  • GET /api/v1/users/refresh_token
  • header：Authorization, X-User
  • 若 token 驗證成功，產出新 JWT，exp 延長一天

Users - profile

  • GET /api/v1/users/profile
  • 回傳 email / username / ability(viewer|editor) / companies (id, name, type, advertisers)
  • 需排除 SuperDSP 白名單格式細項

AdFormatType option

  • GET /api/v1/ad_format_types/option
  • 回傳所有可用格式細項，含 universal_id 等資訊，並依使用者權限過濾

素材 Video/Banner/Pack 建立 API

針對 SuperDSP 素材：

  • 皆需 header：Authorization, X-User
  • form-data body 需帶 source = "superdsp"
  • 建立 pack 時 body 包含 click_urls / text_contents / banner_ids / video_ids / source_id / ad_format_id / ad_format_type_id / vast_feature / video_duration 等欄位
  • 更新 pack API similar to create；若帶 user token 會依 token type 寫入 pack_source 等欄位

    • I/O 單名稱
    • 廣告目的：Brand Building（可選：Brand Building / Drive Traffic / Push Conversion）
    • 選擇產業主/子類型

2️⃣ 建立廣告活動

  1. 點擊「新增廣告活動」
  2. 填寫：
    • 廣告活動名稱
    • 廣告品類：RichMedia（可選：InStream / RichMedia）
    • 廣告走期
    • 預算

3️⃣ 建立廣告組合

  1. 點擊「新增廣告組合」
  2. 填寫：
    • 廣告組合名稱
    • 選擇數據鎖定
    • 選擇格式：如 MIB Flash - MIB Flash Hit Banner

📦 Studio 格式素材包處理情境

✅ 已有素材包（預先於 Studio 建立）

  • 在「指定素材包」欄位中可選擇該 OneAD Studio 素材包
  • 完成其他欄位後儲存完成

❌ 無對應素材包（尚未建立）

  • 「指定素材包」下拉選單找不到目標格式素材包
  • 「新增素材包」按鈕點擊後顯示 message box，點擊確認後另開新分頁並 redirect 到 OneAD Studio

❌ 剛建立但下拉未更新

  • 點擊 SELECT 會重新 fetch 列表，需做 debounce 防止短時間內重複點擊

🧭 素材管理頁導向 STUDIO 建立素材流程

使用左側選單路徑

  1. 點擊左側 MENU BAR「素材管理」
  2. 選擇 TAB「素材包」
  3. 點擊「前往 Studio 建立素材」或「新增素材包」按鈕

▶ 若點擊「前往 Studio 建立素材」

  • 顯示 Message Box：「將以開新視窗方式，帶您前往 STUDIO」
  • 點擊【確認】：
    • 表單抽屜關閉
    • 呼叫 /studio/redirect API 並開啟新視窗
  • OneAD Studio 驗證 token 成功後導入新增畫面
  • 回到 SuperDSP 後需手動重新整理頁面

▶ 若點擊「新增素材包」

  • 顯示素材包表單（名稱、廣告品類、格式、素材上傳與備註）
  • 若選擇 Studio 格式：
    • 顯示 Message Box 並提示前往 OneAD Studio
    • 點擊【確認】 → 另開 Studio 畫面
    • 點擊【取消】 → 留在 SuperDSP 畫面

🔔 注意事項

  • 所有 Studio 另開新視窗 redirect 動作需經過用戶手動確認
  • SuperDSP 無法主動偵測 Studio 中是否已有素材
