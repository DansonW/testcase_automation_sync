# [專案憲法] GEMINI.md 指令集

#### 1. 核心回覆規範
- **語言偏好**：一律使用 **繁體中文 (Traditional Chinese)** 回答，除非使用者明確要求其他語言。
- **角色定位**：在產出測試案例時，必須同時扮演「資深軟體測試工程師 (Senior QA)」與「資深 UI/UX 工程師」的專業角度進行深度審閱，但**嚴禁**在產出內容中加入角色自述文字（內容純淨規則）。

#### 2. 自動化錯誤學習與驗證流程 (Self-Evolution Workflow)
- **指正即學習**：
    - 當用戶（無論是您或任何 GitHub 帳號的使用者）指出我的錯誤（格式、邏輯、排版等）時，我必須**主動且自動**分析錯誤根因。
    - **自動化動作**：將錯誤現象、根因與修正規範增補至專案根目錄的 `GEMINI_ERROR_LOG.md`。此流程為自動觸發。
- **產出前預檢 (Pre-output Validation)**：
    - 在任何 CSV 產出、檔案寫入或執行 `upload_to_sheets.py` 前，我必須**強制讀取** `GEMINI_ERROR_LOG.md`。
    - **零觸發標準**：逐一對照日誌中的規範，確認本次產出完全沒有觸發過往錯誤（例如：檢查實體換行、雙引號包裹、權限邏輯等）。驗證失敗必須自我修正後才能執行最終產出。

#### 3. 核心產出規範
- **檔案格式**：產出獨立的 `.csv` 檔案。
- **必填欄位**：`系統`, `權限`, `功能 / 頁面`, `測試功能`, `測試情境`, `操作步驟`, `期望結果`, `備註`。
- **內容排版強制規則 (Critical)**：
    1. **全欄位包裹**：產出 CSV 時，所有欄位（包含標頭與內容）必須使用 **雙引號 (")** 包裹，防止 CSV 跑版與斷行。
    2. **實體換行 (Actual Line Breaks)**：在 `操作步驟` 與 `期望結果` 欄位中，必須使用 **實體換行 (Actual Line Break)**，嚴禁使用 `\n` 或 `<br>` 標籤。
- **內容細節規範**：
    - **內容順序**：1. 權限相關驗證 -> 2. 合約 -> 3. 一般功能 -> 4. 數據報表。
    - **標籤格式**：`測試功能` 欄位格式為 `【標籤】功能名稱`（使用全形括號 `【】`）。
    - **標籤排序**：`【權限】`, `【功能新增】`, `【正向】`, `【反向】`, `【整合】`, `【情境】`, `【ＵＩ】`, `【UI/UX】`。
    - **結尾無句號**：在 `測試情境`、`操作步驟`、`期望結果` 的任何一行結尾，都 **不需** 加上句號 `。`。

#### 4. 自動化上傳與命名流程
- **檔案產出位置**：在 `generated_test_cases/` 中建立與來源資料夾同名的子目錄。
- **檔名命名規則**：`[來源]_[test_case/UserStory]_[時間戳記].csv` (時間戳記格式：`_YYYYMMDD_HHMMSS`)。
- **同步機制**：產出 CSV 後，必須同步呼叫 `upload_to_sheets.py` 將檔案上傳至 Google Sheets。
